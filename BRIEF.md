## 3. ДИЗАЙН-СИСТЕМА

### Логотип и маскот
- **Логотип** — корова Дурняша, сидящая на кусочке угля (иллюстрация) + «НА УГЛЕ» рукописным шрифтом
- Надпись «НА УГЛЕ» — неровный, как будто написанный углём шрифт
- Логотип адаптируется: полноцветный, чёрно-белый, белый на тёмном фоне
- **Маскот Дурняша** — появляется в разных эмоциях по сайту (довольная, удивлённая, задумчивая). Это голос и лицо бренда.

### Цветовая палитра
> Источник: интерьер выполнен в угольно-чёрном и древесно-коричневом
```
--charcoal:   #1a1a18   // угольно-чёрный — основной фон
--wood:       #5c3d2e   // древесно-коричневый — тепло дерева
--ember:      #e8721c   // жар огня — основной акцент
--amber:      #f5a623   // тёплый янтарь — вторичный акцент
--cream:      #f5f0e8   // крафт / светлый фон секций
--smoke:      #2d2d2a   // тёмно-серый
--ash:        #888880   // вторичный текст
--white:      #ffffff
```

### Типографика
> Основной шрифт проекта — **Courier New Cyr** (моноширинный, эффект печатной машинки)
> Источник: https://fonts-online.ru/fonts/courier-new-cyr

```
Основной (body):   'Courier New Cyr' — моноширинный, typewriter-feel
                   Усиливает ощущение "написанного вручную", аутентичности
Display:           Рукописный шрифт для заголовков — в стиле логотипа
                   Вариант: 'Caveat' (Google Fonts) — близко к логотипу
Акцент:            'Courier New Cyr' bold — для цен, лейблов, кнопок
```

**Принцип пары:** Рукописный display + Courier body = ручная работа + честность без пафоса.

### Визуальный тон и интерьер
- Угольно-чёрный + древесно-коричневый — два основных цвета
- Детали: полевые цветы, бубенчик коровы, керосиновая лампа, мешковина
- Хоспер — скрытая звезда, огненные светильники на входе
- Тёплый, уютный, как погреб у фермера

### UI-инструмент
При генерации компонентов использовать `uipro-cli` с базовым промтом:
```
Create a warm restaurant landing page with vibrant food photography,
menu preview, reservation system, chef story section, and location map.
Use appetizing warm colors.
```
Адаптировать под конкретный блок, сохраняя tone.

- **Superiority Burger** — imperfect stamp типографика

## 4. СТРУКТУРА САЙТА

### БЛОК 0 — ANNOUNCEMENT BAR
**Тонкая полоска над хедером**
- Текст: текущий режим работы (напр. "Сегодня открыты: 12:00 – 23:00")
- Управляется из Orchid: вкл/выкл + текст
- Цвет: --ember на --charcoal
- На мобиле: скрывается или сжимается

---

### БЛОК 1 — HEADER
**Фиксированный, прозрачный → тёмный при скролле**

Содержимое:
- Логотип (спичка + название) — слева
- Навигация — по центру: `Меню`, `О нас`, `Шеф`, `Галерея`, `Контакты`
- CTA кнопка — справа: `Забронировать`

Мобильная версия:
- Логотип слева, гамбургер-меню справа
- При открытии: full-screen оверлей с навигацией

Orchid-управление: нет (статика)

---

### БЛОК 2 — HERO SLIDER
**Fullscreen, 100vh**

Механика:
- Переключение: hard cut (opacity switch без blur, без анимации движения)
- Управление: стрелки влево/вправо (по краям экрана)
- Автопрокрутка: опционально, настраивается в Orchid
- На мобиле: свайп влево/вправо

Каждый слайд содержит:
- Фоновое фото (fullscreen, object-fit: cover)
- Заголовок (крупно, Cormorant Garamond)
- Подзаголовок (меньше, Jost)
- CTA кнопка (текст + ссылка)

Orchid-модель `HeroSlide`:
```
photo        → image upload
title        → string
subtitle     → string (nullable)
cta_text     → string
cta_url      → string
sort_order   → integer
is_active    → boolean
```

---

### БЛОК 3 — МАНИФЕСТ (Концепция)
**Тёмный фон, крупный текст**

Раскладка (десктоп): две колонки
- Левая: крупная фраза-манифест (2–3 слова, Display font)
- Правая: 3–4 предложения о концепции (Body font)

Под текстом — три кнопки:
- `Меню` → открывает PDF
- `Барное меню` → открывает PDF
- `Винная карта` → открывает PDF

Кнопки открывают PDF в новой вкладке.

Orchid-управление (`SiteSettings`):
```
manifesto_headline   → string
manifesto_text       → textarea
menu_pdf             → file upload
bar_menu_pdf         → file upload
wine_card_pdf        → file upload
```

---

### БЛОК 4 — АТМОСФЕРА (Галерея)
**Горизонтальная прокрутка на десктопе, свайп на мобиле**

- Фотографии едут горизонтально
- Каждое фото: ~80vw ширина, полная высота блока (~70vh)
- Без подписей — чистая эмоция
- Лёгкий parallax при скролле (опционально)

Orchid-модель `GalleryPhoto`:
```
photo        → image upload
section      → enum: atmosphere | dishes | team
alt_text     → string
sort_order   → integer
is_active    → boolean
```

- CTA внизу блока: кнопка "Смотреть полное меню" → `/menu` или PDF

```
sort_order    → integer
```

Orchid-модель `MenuCategory`:
```
name          → string
slug          → string
icon          → string (emoji или SVG)
sort_order    → integer
```

### БЛОК 6 — ШЕФ
**Раскладка: фото слева + текст справа (десктоп) / стопка (мобиле)**

Левая часть:
- Большое фото Андрея
- Под фото: карусель с фото кухни и команды в работе (из галереи, section: team)

Правая часть:

Orchid-модель `ChefProfile`:
```
name           → string
position       → string
photo          → image upload
bio_text       → textarea (rich text)

lavolt_note    → string
is_active      → boolean
```

### БЛОК 7 — ШЕФ РЕКОМЕНДУЕТ
**Отдельный блок после шефа**

- 4–6 блюд, выбранных шефом лично
- Карточка: фото + название + короткий комментарий шефа
- Выглядит как личная рекомендация, не каталог
- Визуально отличается от блока хитов (другой layout, акцентный фон)

Управление: в Orchid поле `is_chef_pick → boolean` у MenuItem + поле `chef_comment → string`

### БЛОК 8 — БРОНИРОВАНИЕ
**Тёмный акцентный фон (--ember или --smoke)**

Форма:
- Имя
- Телефон
- Дата и время
- Количество гостей
- Комментарий (необязательно)
- Кнопка "Отправить заявку"

После отправки:
- Уведомление на email (Laravel Mail + Queue)
- Уведомление в Telegram (опционально, через бота)
- Пользователю: success-сообщение на странице (без перезагрузки)

Orchid-экран `ReservationScreen`:
- Список заявок с фильтром по дате
- Статусы: Новая / Подтверждена / Отменена
- Кнопка звонка из списка (tel: ссылка)

Валидация (FormRequest):
```
name     → required, string, max:100
phone    → required, string
date     → required, date, after:today
guests   → required, integer, min:1, max:50
comment  → nullable, string, max:500
```

### Важные UX-принципы для Orchid

1. **Все переключатели блюд** ("В наличии", "Хит", "Рекомендует шеф") доступны прямо из списка — без входа в карточку
2. **Drag-and-drop сортировка** для слайдов hero и фото галереи
3. **Превью изображения** сразу после загрузки
4. **Подтверждение** перед удалением любого элемента
5. **Автосохранение черновика** при редактировании длинных текстов
6. **Breadcrumbs** на всех экранах — чтобы не потеряться
